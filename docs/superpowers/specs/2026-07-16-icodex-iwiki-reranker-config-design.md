# icodex iwiki reranker config design

## 1. Objective

Update icodex's built-in `iwiki-mcp` Codex MCP wiring so it supports the newer upstream `iwiki-mcp` environment surface, including reranker and search tuning, while making Codex project binding deterministic through generated `IWIKI_PROJECT_DIR`.

This design is intentionally narrow: runtime support and `.codex_config.example` are updated, but the live `.codex_config` file is not changed because it is user-owned and may contain secrets.

## 2. Acceptance (from intent)

- Generated per-home Codex `config.toml` includes the newer iwiki server variables only when the matching `ICODEX_IWIKI_*` values are set in `.codex_config`.
- Generated per-home Codex `config.toml` includes `IWIKI_PROJECT_DIR` pointing at the real project root without requiring the user to configure it manually in `.codex_config`.
- In Codex sessions, `wiki_status` reports the intended project directory and the expected project `read` and `write` binding.
- Reranker support can be enabled through `.codex_config`, and when it is unset, iwiki search keeps the previous non-reranked behavior.
- `.codex_config.example` documents the current iwiki configuration surface, including reranker-related variables and generated project-dir behavior.

Health metrics:

- `IWIKI_LLM_KEY` remains secret and is not written literally into generated Codex `config.toml`, tracked files, docs, tests, or examples.
- Codex launch does not fail when iwiki required settings are incomplete; iwiki wiring continues to warn and skip instead.
- Unset optional iwiki variables are omitted from generated TOML so upstream server defaults still apply.
- Existing project-root `.iwiki.toml` files remain user-owned and are not overwritten.
- Focused iwiki Bash tests remain green, especially wiring, binding, and env-mapping tests.

## 3. Requirements

### 3.1 Runtime variable support

`ensure_iwiki_wiring` must continue to generate a managed `[mcp_servers.iwiki]` block in each per-home Codex `config.toml`. The block must support these optional passthrough variables through the existing `ICODEX_IWIKI_*` pattern:

- `ICODEX_IWIKI_EMBED_MODEL` -> `IWIKI_EMBED_MODEL`
- `ICODEX_IWIKI_EMBED_DIMENSIONS` -> `IWIKI_EMBED_DIMENSIONS`
- `ICODEX_IWIKI_TOP_K` -> `IWIKI_TOP_K`
- `ICODEX_IWIKI_SCORE_THRESHOLD` -> `IWIKI_SCORE_THRESHOLD`
- `ICODEX_IWIKI_SEARCH_MODE` -> `IWIKI_SEARCH_MODE`
- `ICODEX_IWIKI_RERANK_MODEL` -> `IWIKI_RERANK_MODEL`
- `ICODEX_IWIKI_GRAPH_DEPTH` -> `IWIKI_GRAPH_DEPTH`
- `ICODEX_IWIKI_SEED_TOP_K` -> `IWIKI_SEED_TOP_K`
- `ICODEX_IWIKI_BFS_TOP_K` -> `IWIKI_BFS_TOP_K`
- `ICODEX_IWIKI_SEED_THRESHOLD` -> `IWIKI_SEED_THRESHOLD`
- `ICODEX_IWIKI_WRITE_SEED_THRESHOLD` -> `IWIKI_WRITE_SEED_THRESHOLD`
- `ICODEX_IWIKI_CHAT_MODEL` -> `IWIKI_CHAT_MODEL`
- `ICODEX_IWIKI_CHUNK_SIZE` -> `IWIKI_CHUNK_SIZE`
- `ICODEX_IWIKI_CHUNK_OVERLAP` -> `IWIKI_CHUNK_OVERLAP`
- `ICODEX_IWIKI_SUMMARY_MAX_CHARS` -> `IWIKI_SUMMARY_MAX_CHARS`

Definition of done: a generated test config includes exactly the set optional lines for these variables and omits unset optional lines.

### 3.2 Generated project directory

`ensure_iwiki_wiring` must write `IWIKI_PROJECT_DIR` to `[mcp_servers.iwiki.env]` using `$ICODEX_PROJECT_ROOT`. This value is generated by icodex and is not user-configured through `.codex_config`.

Definition of done: a focused test sets `ICODEX_PROJECT_ROOT` to a synthetic project directory and the generated TOML contains `IWIKI_PROJECT_DIR = "<that directory>"`.

### 3.3 Required-tier guard

The required tier for writing the iwiki region must include:

- `ICODEX_IWIKI_COMMAND`, or `command -v iwiki-mcp`
- `ICODEX_IWIKI_BASE_DIR`
- `ICODEX_IWIKI_LLM_BASE_URL`
- `ICODEX_IWIKI_LLM_KEY`, or ambient `IWIKI_LLM_KEY`
- `ICODEX_PROJECT_ROOT`

If any required value is missing, `ensure_iwiki_wiring` must log a warning, skip writing the iwiki region, and return 0.

Definition of done: focused tests cover missing LLM base URL, missing key, and missing project root as non-failing no-region cases.

### 3.4 Secret handling

`IWIKI_LLM_KEY` must continue to be forwarded through `env_vars = ["IWIKI_LLM_KEY"]`. It must not be written into `[mcp_servers.iwiki.env]` or any tracked example.

Definition of done: focused tests assert the synthetic secret is absent from generated TOML.

### 3.5 `.codex_config.example`

`.codex_config.example` must document:

- required iwiki settings;
- optional command override;
- optional embedding, chat, search, rerank, graph, seed, and indexing settings;
- generated `IWIKI_PROJECT_DIR` behavior;
- the fact that `ICODEX_IWIKI_PROJECT_DIR` is intentionally unsupported.

Definition of done: the example includes commented keys for supported `ICODEX_IWIKI_*` values and contains no real secret.

## 4. Architecture

### 4.1 Existing boundary

The existing `lib/iwiki/iwiki.sh` module remains the owner of Codex MCP TOML generation. No new module is needed.

The existing `lib/config/env.sh` parser remains the owner of `.codex_config` key filtering. It already accepts `ICODEX_*` keys and rejects raw `IWIKI_*` keys. That behavior remains.

### 4.2 Optional variable list

`_IWIKI_OPTIONAL_VARS` becomes the single authoritative list of optional non-secret iwiki settings that icodex forwards. Extending this list is enough to add a new `ICODEX_IWIKI_*` passthrough variable when upstream adds a new non-secret environment variable.

`IWIKI_PROJECT_DIR` is not part of `_IWIKI_OPTIONAL_VARS` because it is generated from `ICODEX_PROJECT_ROOT`, not from `.codex_config`.

### 4.3 Region body function

`_iwiki_region_body` changes from:

```bash
_iwiki_region_body() { # <command> <base_dir> <llm_base_url>
```

to:

```bash
_iwiki_region_body() { # <command> <base_dir> <llm_base_url> <project_dir>
```

The generated body shape becomes:

```toml
[mcp_servers.iwiki]
command = "<resolved command>"
env_vars = ["IWIKI_LLM_KEY"]
[mcp_servers.iwiki.env]
IWIKI_BASE_DIR = "<base dir>"
IWIKI_LLM_BASE_URL = "<LLM base URL>"
IWIKI_PROJECT_DIR = "<project root>"
<optional IWIKI_* lines for set ICODEX_IWIKI_* values>
```

`command` and `env_vars` stay before `[mcp_servers.iwiki.env]` so they remain parent-table keys, not env-subtable keys.

## 5. Data Flow

1. `load_config` reads `.codex_config` and exports allowed `ICODEX_*` variables.
2. `apply_iwiki_env` maps `ICODEX_IWIKI_LLM_KEY` to ambient `IWIKI_LLM_KEY` unless ambient `IWIKI_LLM_KEY` already exists.
3. `resolve_codex_home` and setup code set `ICODEX_PROJECT_ROOT`.
4. `ensure_iwiki_wiring` resolves command, base dir, LLM base URL, secret presence, and project root.
5. If all required values resolve, `ensure_iwiki_wiring` rewrites the managed iwiki region in the per-home `config.toml`.
6. Codex spawns `iwiki-mcp`, forwards `IWIKI_LLM_KEY` by `env_vars`, and passes literal non-secret settings through `[mcp_servers.iwiki.env]`.
7. `iwiki-mcp` resolves `.iwiki.toml` from `IWIKI_PROJECT_DIR`, so `wiki_status.project_dir`, `read`, and `write` reflect the launched project, not `CODEX_HOME`.

## 6. Error Handling

- Missing command, base dir, LLM base URL, key, or project root: log a warning, do not write a partial iwiki region, return 0.
- Missing optional variable: omit it; upstream server default applies.
- Manual `ICODEX_IWIKI_PROJECT_DIR`: do not document it, do not add special handling for it, and let generated `IWIKI_PROJECT_DIR` win.
- Existing project `.iwiki.toml`: preserve existing `ensure_iwiki_binding` behavior; this change does not overwrite it.
- Secret leakage: treat any generated TOML containing the literal secret as a test failure and implementation blocker.

## 7. Tests

Focused test updates:

- `tests/test_iwiki_wiring.sh`
  - Set `ICODEX_PROJECT_ROOT` and assert generated `IWIKI_PROJECT_DIR`.
  - Set new optionals such as `ICODEX_IWIKI_RERANK_MODEL`, `ICODEX_IWIKI_SEARCH_MODE`, `ICODEX_IWIKI_SEED_TOP_K`, `ICODEX_IWIKI_BFS_TOP_K`, `ICODEX_IWIKI_SEED_THRESHOLD`, `ICODEX_IWIKI_WRITE_SEED_THRESHOLD`, and `ICODEX_IWIKI_CHAT_MODEL`; assert matching `IWIKI_*` lines.
  - Assert unset new optionals are absent.
  - Assert missing `ICODEX_PROJECT_ROOT` is a no-region non-failing guard case.
  - Keep existing assertions for idempotence, stale region stripping, auto-detected command, missing URL/key guard, and secret not written.
- `tests/test_iwiki_env.sh`
  - Assert `ICODEX_IWIKI_RERANK_MODEL` loads as a wrapper variable.
  - Assert raw `IWIKI_RERANK_MODEL` remains ignored when present in `.codex_config`.

## 8. Documentation

- `.codex_config.example` documents the updated iwiki configuration surface.
- The icodex iwiki page `iwiki-mcp-integration` is updated after implementation to describe the new optional variables and generated project-dir behavior.
- `docs/TODO.md` is updated by the chain gate as the task moves through spec, plan, and result.

## 9. Non-Goals

- Do not modify live `.codex_config`.
- Do not change real secret values.
- Do not change `iwiki-mcp` itself.
- Do not add an iwiki enable/disable flag.
- Do not switch Codex MCP invocation from `command` to `args`.
- Do not remove the existing `.iwiki.toml` symlink behavior; it remains harmless and preserves compatibility.
