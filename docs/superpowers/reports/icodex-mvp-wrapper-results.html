Спецификация bash-обёртки icodex над OpenAI Codex CLI — MVP-ядро по примеру проекта iclaude.

1. Резюме спецификации

Требования по разделам

§Требование
1icodex — самодостаточный launcher Codex CLI: изоляция установки, изоляция состояния, proxy, прозрачный запуск
1Изоляция установки: пиннутый статичный musl-бинарь в проекте, без global-install и без Node
1Изоляция состояния: CODEX_HOME → проектный каталог .codex-isolated/
1Proxy: экспорт HTTPS_PROXY/HTTP_PROXY в окружение codex
1Прозрачный запуск: проброс неузнанных аргументов в реальный codex
2Структура — standalone-зеркало паттерна iclaude, только MVP-модули, ноль связности
3.1icodex.sh оркестрирует: source → парс args → branch → setup → exec
3.2binary/detect.sh: uname → имя release-ассета (linux/darwin × x86_64/aarch64)
3.3binary/install.sh: скачать, sha256-верификация (tamper guard), распаковать в bin/
3.4binary/lockfile.sh: .codex-lockfile.json {version, asset, sha256}, в git
3.5config/isolated.sh: export CODEX_HOME; auth passthrough — ключи wrapper не хранит
3.6proxy/proxy.sh: --proxy сохраняет url (chmod 600), export *_PROXY; --no-proxy
3.7launcher/launch.sh: exec codex "$@"
4CLI: --proxy / --no-proxy / --clear / --update / --install / --version / --help / passthrough
5Ошибки: отсутствие deps, неизвестная платформа, sha-mismatch (стоп), сеть/404
6Тесты: detect / lockfile / install / proxy / isolated / smoke

Критерии успеха (Success Criteria)

#Критерий
1На чистой машине ./icodex.sh скачивает пиннутый бинарь и верифицирует sha256
2CODEX_HOME изолирован в .codex-isolated/
3codex стартует в полностью изолированном окружении проекта
4--proxy <url> наглядно направляет трафик codex через прокси

2. Схемы решения и зависимостей

Схема решения (C4 — компоненты и потоки)

Поток: флаги юзера → env (CODEX_HOME + *_PROXY) → exec codex наследует env → codex читает изолированные config.toml/auth.json.

Граф зависимостей (A→B = A зависит от B)

Граф ацикличный: оркестратор зависит от модулей; install — от detect/lockfile/validation; launcher — от config и install. Циклов нет.
