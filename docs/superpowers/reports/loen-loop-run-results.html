Проверка intent — 2026-07-08

Интент фиксирует переход LoEn от ручной цепочки после `loop-start` к guided workflow: старт собирает требования, выбирает режим, утверждает plan, затем передаёт execution в отдельный `loen:loop-run <topic>`.

Главный риск — не превратить automation в обход safety gates. Документ сохраняет `docs/loen/<topic>/` как источник истины, оставляет manual skills совместимыми и требует policy/evidence для delivery, governance auto-fix и governance merge/release.

Проверка spec — 2026-07-08

Спека проектирует explicit state machine для нового `loen:loop-run ` и переносит human planning gate в `loen:loop-start`. Пользователь выбирает `delivery` или `governance`, затем для governance выбирает subtype: `report-only`, `auto-fix`, `merge-release`.

Проверка spec подтверждает: intent outcomes покрыты, manual LoEn path остаётся совместимым, `merge-release` включён в текущий design, а safety gates остаются обязательными.
