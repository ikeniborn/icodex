Проверка intent — 2026-07-08

Обзор

Интент фиксирует доверенную цель: добавить opt-in PII proxy для Codex OpenAI API traffic, чтобы критичная персональная информация и секреты не уходили в OpenAI cloud без masking.

Langfuse capture выделен как отдельный доверенный режим анализа промптов на пользовательском сервере. OpenAI-compatible provider routing явно не входит в scope.

Проверка spec — 2026-07-08

Обзор

Спека фиксирует выбранную архитектуру: opt-in PII proxy запускается локально, а Codex направляется в него через one-run openai_base_url. Persistent provider/config mutation не требуется.

Основной safety contract: без PII обычный запуск не меняется; с PII включён fail-secure; masking precision-first через rules по умолчанию и optional nlp с EN/RU model install/update.
