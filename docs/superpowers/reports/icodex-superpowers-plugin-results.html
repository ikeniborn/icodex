Цель: вшить фреймворк навыков Superpowers в изолированный Codex CLI icodex как git-доставляемый плагин. Установленный плагин, пользовательские skills и вся обвязка едут в репозитории; свежий клон не требует установки плагина — на --install тянется только бинарь (через прокси).

Ключевые проектные решения

РешениеОбоснование (эмпирически проверено на codex-cli 0.142.2)
Коммитим CACHE-дерево, а не дубль marketplace-sourcesource может указывать прямо на cache-копию (в ней валидный .codex-plugin/plugin.json) → один коммит ~2 MB вместо двух
Launcher переписывает source на запускеcodex валидирует source-путь на каждом запуске; перенос репо ломает загрузку; rewrite строки чинит (доказано), а -c override — нет
Каноническое имя marketplace superpowersу marketplace add нет --name, имя авто-выводится (superpowers-dev) → vendor-скрипт нормализует имя/каталог
features.multi_agent=true + bypass_hook_trust=truesubagent-skills требуют multi_agent; hook требует trust → bypass для не-интерактивного SessionStart
