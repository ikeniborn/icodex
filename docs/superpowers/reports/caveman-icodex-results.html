Требования по разделам

Раздел / требованиеСуть
ProblemТерсный вывод модели в Codex (экономия output-токенов ~65–75% на prose-heavy ходах) при ~0 input-overhead самого caveman-механизма.
Terminology новыйShip default = caveman off, когда ICODEX_CAVEMAN_MODE не задан (out-of-the-box). Active launch mode = значение ICODEX_CAVEMAN_MODE на старте (lite|full|ultra), подставляется в AGENTS.md-блок при рендере.
SC1При ICODEX_CAVEMAN_MODE=full вывод Codex терсный (caveman-стиль).
SC2Пока текущий режим сессии == active launch mode, per-turn hook инжектит 0 токенов (вся стоячая инструкция в prompt-cached AGENTS.md).
SC3/caveman lite|full|ultra|off (+ stop caveman / normal mode) переключает режим в середине сессии.
SC4Файлы целевого проекта не трогаются — caveman живёт в icodex-owned CODEX_HOME.
SC5Ship default = off; caveman активируется только когда ICODEX_CAVEMAN_MODE задан.
BackgroundТочки расширения icodex: глобальный $CODEX_HOME/AGENTS.md; lifecycle-хуки (SessionStart, UserPromptSubmit) через top-level config.toml или hooks.json; icodex уже поставляет общий .codex-isolated/hooks.json (PreToolUse secret-guards), симлинк в каждый home, авто-обнаружение из $CODEX_HOME/hooks.json — caveman-хук подключается в этот же файл; обход repo-local hook-бага.
DecisionsScope = always-on once enabled + in-session switch/off; механизм hybrid (cached AGENTS.md + лёгкий hook); source hybrid (правила из upstream SKILL.md, hook нативный); ship default off; один UserPromptSubmit hook без SessionStart; язык хука python3.
Layer 1 — AGENTS.md base blockСтатический кэшируемый блок (persona + правила сжатия, mode table, исключения авто-ясности, языковое правило, /caveman-контракт). <2 KiB, delimited region, идемпотентный rewrite. Active launch mode подставляется при рендере.
Layer 2 — UserPromptSubmit hookРегистрируется через per-project home hooks.json (авто-обнаружение из $CODEX_HOME/hooks.json). Один python3-скрипт (stdlib): lazy-init режима из state-файла; /caveman switch → запись + additionalContext; current == active launch mode → пустой stdout (0 токенов); deviation → ACTIVE MODE reminder; off → disable line. State по session_id под $CODEX_HOME/.caveman/.
Data flowicodex.sh launch → ensure_caveman_wiring → (1) AGENTS.md регион; (2) $CODEX_HOME/hooks.json: mode set → реальный файл = merge(shared secret-guards, caveman UserPromptSubmit → python3 caveman-hook.py); off/unset → симлинк на shared hooks.json. Codex turn → UserPromptSubmit → caveman-hook.py.
Asset layout.codex-isolated/hooks/caveman-hook.py (рядом с block-secrets.py), .codex-isolated/caveman/agents-block.md, scripts/vendor-caveman.sh, lib/caveman/caveman.sh (ensure_caveman_wiring: рендер AGENTS.md регион + рендер/merge home hooks.json).
Config and wiringICODEX_CAVEMAN_MODE в .codex_config; ensure_caveman_wiring после ensure_superpowers_wiring; две идемпотентные операции (AGENTS.md регион; home hooks.json merge/симлинк); setup_codex_home сначала (_link_shared hooks.json), caveman переопределяет только при включении; bypass_hook_trust=true.
Token-overhead accounting (goal 2)Таблица состояний: == launch mode → 0; switched → ~10–20 ток; off → ~10 ток.
Teststest_caveman_wiring.sh + test_caveman_hook.sh. Без сети, temp-dirs.
Documentation (iwiki)Новый docs/wiki/caveman.md; обновить config.md, architecture.md, command.md; iwiki-ingest + iwiki-lint.
Out of scope (v1)Stats/statusline; wenyan-*/commit/review/compress режимы; verbatim-порт upstream Node-хуков.
