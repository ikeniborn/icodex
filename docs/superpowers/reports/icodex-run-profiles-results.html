Требования по разделам

РазделТребование
GoalICODEX_MODE — полный профиль запуска с двумя headline-режимами (run-with-prompts и full no-stop); .git writable в каждом writable-режиме.
Problem-1Сегодня apply_sandbox_mode пишет только sandbox_mode; approval_policy/default_permissions зашиты в шаблон — нет способа получить full no-stop.
Problem-2.git перемонтируется read-only при активном managed-профиле даже на danger-full-access; git-записи падают.
BackgroundДвухслойная модель codex (sandbox layer + managed-permission layer), строже выигрывает; approval_policy ортогонален обоим слоям.
Decisions · модельОдин пресет ICODEX_MODE бандлит 3 настройки; гранулярные env переопределяют отдельные поля; новых CLI-флагов нет.
Decisions · пресеты4 пресета: ro / safe / full-ask / full-auto; дефолт full-ask (воспроизводит шаблон + теперь .git-writable).
Decisions · full-autofull-auto убирает default_permissions (managed layer off, эквив. --dangerously-bypass-approvals-and-sandbox), эмитит log_warn.
Decisions · .git fixДобавить ".git/" = "write" в :workspace_roots профилей dev-safe и ssh-on-request, идемпотентно каждый запуск (миграция старых home).
Decisions · precedencedefaults(full-ask) < ICODEX_MODE < granular (ICODEX_SANDBOX | ICODEX_APPROVAL | ICODEX_PERMISSIONS); --full-access по-прежнему форсит только sandbox-поле.
Decisions · валидацияНевалидный ICODEX_MODE или любое невалидное гранулярное значение → log_error + ненулевой код.
Architecture · resolve_modeЧистый резолвер: echo тройки sandbox approval permissions; старт с пресета, override полей из env; permissions может быть сентинелом none.
Architecture · TOML-хелперы_upsert_toml_toplevel без изменений; новый _remove_toml_toplevel — идемпотентно удаляет top-level ключ (byte-identical при отсутствии).
Architecture · apply_modeОркестратор (заменяет apply_sandbox_mode на run-path): resolve → upsert sandbox/approval → ветвь по permissions → log_warn для danger / none.
Architecture · ensure_git_writableВ permissions.sh: идемпотентно апсертит ".git/" = "write" под [permissions.<profile>.filesystem.":workspace_roots"].
Architecture · run-pathapply_sandbox_mode || exit 1 → apply_mode || exit 1 в icodex.sh; порядок относительно trust/binary-grant не меняется.
Architecture · шаблонШаблон config.toml: сохранить full-ask дефолт, добавить .git grant в оба профиля, обновить комментарии под ICODEX_MODE.
Error handlingНевалидное значение → выход 1 до запуска codex; _remove_toml_toplevel на отсутствующем ключе — no-op; ensure_git_writable создаёт секцию при отсутствии; повторный запуск идемпотентен.
TestingНовый test_mode.sh + расширение test_install.sh: resolve_mode, apply_mode, _remove_toml_toplevel, ensure_git_writable.
Documentation.codex_config.example (блок ICODEX_MODE), docs/wiki/config.md (секция Run mode + iwiki), README.md и print_help.
Global constraintsbash + awk; set -euo pipefail в lib, set -uo pipefail в тестах; 2-space indent; ICODEX_ префикс; CODEX_HOME/proxy/binary/trust не трогаются.
Out of scope (YAGNI)Новые CLI-флаги; кастомный no-prompt профиль с защитой секретов; per-project ICODEX_MODE; запись в иные protected paths кроме .git; переписывание исторических home.

1resolve_mode выдаёт точную тройку для каждого пресета и при переопределениях полей; невалидные значения → return 1 + log_error.
2apply_mode записывает sandbox_mode/approval_policy; full-auto удаляет default_permissions; non-none пишет его; warnings для danger / none.
